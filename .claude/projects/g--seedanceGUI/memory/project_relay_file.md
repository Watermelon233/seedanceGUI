## 接力文件维护规范

**规则**: 只使用一个接力文件，通过覆盖更新维护，避免文件持续增长

**Why**: 用户反馈："记录下来 不要每次新开一个接力文件 通过覆盖填写同一个文件 避免文件持续增长"

**How to apply**:
1. **统一文件**: 只使用 `接力文件.md` 一个文件
2. **覆盖更新**: 新内容覆盖旧内容，保留有价值的历史信息
3. **时间倒序**: 最新内容在文件顶部，便于快速查看
4. **定期清理**: 文件过大时归档历史内容
5. **更新位置**: 文件顶部显示"最后更新"时间戳

**文件结构**:
- 顶部: 项目状态（最后更新时间、当前状态）
- 中部: 已完成工作、当前问题、待完善功能
- 底部: 工作流程规范、技术决策记录
