## 🔧 技术决策记录

### Mock API 设计原则
1. **零侵入**: videoService 不需要知道 mock 的存在
2. **状态持久化**: localStorage 存储，刷新页面不丢失
3. **容错性**: localStorage 失败自动降级到内存模式
4. **生产安全**: 构建时强制校验，防止误发布

### 环境变量控制
```bash
# .env (开发测试)
VITE_MOCK_API=true

# 生产环境会自动禁用
```

### 2. 任务列表管理功能 📋

**需求描述**:
- 任务列表通过 API key 关联（区分不同用户）
- 记录任务创建时间、完成时间
- 支持下载功能
- 已提交未完成的任务需要定时轮询查询状态
- 根据查询结果自动切换视频状态

**技术方案**:

1. **数据结构** (IndexedDB + localStorage):
   ```typescript
   interface TaskRecord {
     id: string;              // 任务ID
     apiKey: string;          // 关联的API key（脱敏存储）
     model: string;           // 使用的模型
     prompt: string;          // 提示词
     status: 'pending' | 'processing' | 'completed' | 'failed';
     videoUrl: string | null;
     createdAt: number;       // 创建时间戳
     completedAt: number | null;
     thumbnail?: string;      // 缩略图
   }
   ```

2. **定时轮询机制**:
   - 使用 `setInterval` 每 5-10 秒查询一次未完成任务
   - 页面可见性优化：页面隐藏时暂停轮询
   - 最大轮询次数限制（避免无限轮询）

3. **UI 组件**:
   - `TaskListPage.tsx` - 任务列表页面
   - `TaskItem.tsx` - 单个任务卡片
   - 集成到左侧导航

**涉及文件**:
- `src/pages/TaskListPage.tsx` (新建)
- `src/services/taskListService.ts` (新建)
- `src/types/index.ts` (扩展类型)

### 3. 图床功能 (tempfile.org) 🖼️

**需求描述**:
- 使用 https://tempfile.org/api 上传图片/视频/音频
- 用户可选择使用 base64 上传或 URL 上传

**API 参考** (需要验证):
```http
POST https://tempfile.org/api/upload
Content-Type: multipart/form-data

file: <binary>
or
url: <string>
```

**技术方案**:

1. **双模式上传**:
   - **Base64 模式**: 用户上传本地文件，转换为 base64 后发送
   - **URL 模式**: 用户提供外部 URL，服务器拉取后托管

2. **UI 组件**:
   - 文件选择器（支持图片、视频、音频）
   - Base64/URL 切换开关
   - 上传进度显示
   - 上传成功后返回托管 URL

3. **集成点**:
   - SingleTaskPage 的图片上传区域
   - 替代当前的本地文件预览模式

**涉及文件**:
- `src/services/imageHostingService.ts` (新建)
- `src/pages/SingleTaskPage.tsx` (修改)
- `src/components/ImageUploader.tsx` (新建)

**注意事项**:
- 需要验证 tempfile.org API 的具体参数格式
- 考虑文件大小限制
- 处理上传失败的情况
