**功能说明**:
- 通过 `VITE_MOCK_API=true` 环境变量启用
- localStorage 持久化任务状态，刷新页面不丢失
- 内存 fallback 支持，隐私模式下自动切换
- 5 分钟自动清理过期任务
- 右下角 Mock 控制面板可查看/清理任务
- 顶部黄色横幅提示测试模式
- 生产构建自动禁用保护

**使用方法**:
1. 在 `.env` 中设置 `VITE_MOCK_API=true`
2. 重启开发服务器
3. 访问 `/generate` 页面测试

## 🔧 技术决策记录

### Mock API 设计原则
1. **零侵入**: videoService 不需要知道 mock 的存在
2. **状态持久化**: localStorage 存储，刷新页面不丢失
3. **容错性**: localStorage 失败自动降级到内存模式
4. **生产安全**: 构建时强制校验，防止误发布

### 环境变量控制
```bash
# .env (开发测试)
VITE_MOCK_API=true

# 生产环境会自动禁用
```
