# Seedance 2.0 扩展支持音频和视频参考

## 📋 需求分析

### 当前状态
- ✅ 支持4种视频生成模式
- ✅ 支持图片参考（image_url）
- ❌ 不支持音频参考（audio_url）
- ❌ 不支持视频参考（video_url）

### 模型能力
根据MODEL_OPTIONS描述：
- "音视频图均可参考"
- Seedance 2.0是全能模型

## 🎯 扩展方案

### 1. content数组支持多种媒体类型

```typescript
// 当前只支持图片
content = [
  { type: 'text', text: '...' },
  { type: 'image_url', image_url: { url: '...' } }
]

// 扩展后支持多种媒体
content = [
  { type: 'text', text: '...' },
  { type: 'image_url', image_url: { url: '...' } },
  { type: 'audio_url', audio_url: { url: '...' } },
  { type: 'video_url', video_url: { url: '...' } }
]
```

### 2. 新的5种生成模式

扩展4种模式为5种模式：

1. **文生视频** - 纯文本
2. **图生视频-首帧** - 1张图片
3. **图生视频-首尾帧** - 2张图片
4. **图生视频-参考图** - 多张图片
5. **混合参考模式** - 🆕 图片+音频+视频混合

### 3. 混合参考模式的媒体限制

```typescript
{
  [MediaMode.MIXED]: {
    maxImages: 9,
    maxAudio: 3,      // 最多3个音频
    maxVideos: 2,     // 最多2个视频
    defaultRatio: '16:9',
    useRole: true,
    roleType: 'reference'
  }
}
```

### 阶段3：content数组构建

```typescript
async function buildContentArray(
  text: string,
  mediaFiles: MediaFile[]
): Promise<any[]> {
  const content: any[] = [];
  
  // 添加文本
  content.push({ type: 'text', text });
  
  // 添加媒体文件
  for (const file of mediaFiles) {
    const base64 = await fileToBase64(file.file);
    
    switch (file.type) {
      case MediaType.IMAGE:
        content.push({
          type: 'image_url',
          image_url: { url: base64 },
          role: 'reference_image'
        });
        break;
      
      case MediaType.AUDIO:
        content.push({
          type: 'audio_url',
          audio_url: { url: base64 }
        });
        break;
      
      case MediaType.VIDEO:
        content.push({
          type: 'video_url',
          video_url: { url: base64 }
        });
        break;
    }
  }
  
  return content;
}
```
