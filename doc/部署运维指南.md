## 2. 本地开发部署

### 2.1 安装依赖

```bash
# 1. 克隆项目
git clone https://github.com/wwwzhouhui/seedance2.0.git
cd seedance/seedance-temp

# 2. 安装前端依赖
npm install

# 3. 安装后端依赖
cd server
npm install
cd ..

# 4. 安装 Chromium（用于绕过即梦反爬）
npx playwright-core install chromium
```

### 2.2 配置环境变量

```bash
# 复制环境变量模板
cp .env.example .env
```

编辑 `.env` 文件：

```env
# ==================== 基础配置 ====================
# Express 后端端口
PORT=3001

# 默认即梦 SessionID（可选，用户可在设置中配置）
VITE_DEFAULT_SESSION_ID=your_session_id_here

# ==================== 邮件服务 (SMTP) ====================
# SMTP 服务器配置
SMTP_HOST=smtp.163.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=your_email@163.com
SMTP_PASS=your_smtp_password
SMTP_FROM=seedance@example.com
SMTP_FROM_NAME=Seedance 2.0
SMTP_TLS_REJECT_UNAUTHORIZED=true

# ==================== 应用配置 ====================
# 每日签到积分
DAILY_CHECK_IN_CREDITS=2

# 新用户初始积分
NEW_USER_CREDITS=10
```

### 2.3 启动开发服务

```bash
# 方式一：同时启动前端和后端
npm run dev

# 方式二：分别启动
npm run dev:client   # 前端 :5173
npm run dev:server   # 后端 :3001
```

访问地址：
- 前端：http://localhost:5173
- 后端 API：http://localhost:3001/api

### 2.4 开发工具

```bash
# 前端类型检查
npx tsc --noEmit

# 构建前端
npm run build

# 生产启动
npm start
```

```bash

# 重启服务
docker compose restart

# 重新构建
docker compose up -d --build

# 进入容器
docker compose exec app bash
```

### 3.2 使用 Dockerfile

#### 3.2.1 构建镜像

```bash
docker build -t seedance2.0:latest .
```

#### 3.2.2 运行容器

```bash
docker run -d \
  --name seedance2.0 \
  -p 3000:3000 \
  -p 3001:3001 \
  -v $(pwd)/data:/app/data \
  -e PORT=3001 \
  -e SMTP_HOST=smtp.163.com \
  -e SMTP_PORT=465 \
  -e SMTP_USER=your_email@163.com \
  -e SMTP_PASS=your_password \
  seedance2.0:latest
```

## 4. 生产环境部署

### 4.1 直接部署

#### 4.1.1 构建项目

```bash
# 安装依赖
npm run install:all

# 安装 Chromium
npx playwright-core install chromium

# 构建前端
npm run build
```

#### 4.1.2 配置 PM2（推荐）

安装 PM2：

```bash
npm install -g pm2
```

创建 `ecosystem.config.js`：

```javascript
module.exports = {
  apps: [{
    name: 'seedance2.0',
    script: './server/index.js',
    cwd: './seedance-temp',
    instances: 1,
    exec_mode: 'fork',
    env: {
      NODE_ENV: 'production',
      PORT: 3001
    },
    error_file: './logs/err.log',
    out_file: './logs/out.log',
    log_date_format: 'YYYY-MM-DD HH:mm:ss Z',
    merge_logs: true,
    autorestart: true,
    max_restarts: 10,
    min_uptime: '10s'
  }]
};
```

启动服务：

```bash
# 启动应用
pm2 start ecosystem.config.js

# 查看状态
pm2 status

# 查看日志
pm2 logs seedance2.0

# 设置开机自启
pm2 startup
pm2 save
```

### 4.2 Nginx 反向代理

#### 4.2.1 Nginx 配置

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # API 代理
    location /api/ {
        proxy_pass http://localhost:3001/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 文件上传大小限制
        client_max_body_size 50M;
    }

    # 视频代理
    location /api/video-proxy {
        proxy_pass http://localhost:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;

        # 流式传输
        proxy_buffering off;
        proxy_cache off;
    }
}
```

#### 4.2.2 HTTPS 配置（Let's Encrypt）

```bash
# 安装 certbot
sudo apt install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo certbot renew --dry-run
```

## 7. 故障排查

### 7.1 常见问题

#### 问题 1：服务无法启动

**现象**：启动后立即退出

**排查**：
```bash
# 查看日志
pm2 logs seedance2.0 --lines 50

# 检查端口占用
netstat -tulpn | grep 3001
```

**解决**：
- 检查 `.env` 配置是否正确
- 检查端口是否被占用
- 检查数据库文件权限

#### 问题 2：视频生成失败

**现象**：生成任务一直处于 processing 状态

**排查**：
```bash
# 查看即梦 Session 配置
SELECT * FROM jimeng_session_accounts;

# 查看生成日志
pm2 logs seedance2.0 | grep "提交"
```

**解决**：
- 确认 SessionID 有效
- 检查即梦平台账号积分
- 检查浏览器代理是否正常

#### 问题 3：邮件发送失败

**现象**：注册/登录时收不到验证码

**排查**：
```bash
# 查看 SMTP 配置
SELECT * FROM system_config WHERE key LIKE '%smtp%';
```

**解决**：
- 确认 SMTP 服务器地址和端口
- 检查邮箱授权码（非登录密码）
- 确认服务器防火墙允许 SMTP 端口

#### 问题 4：下载失败

**现象**：视频无法下载到本地

**排查**：
```bash
# 检查下载目录权限
ls -la data/videos/

# 查看下载服务日志
pm2 logs seedance2.0 | grep "下载"
```

**解决**：
- 确认下载目录存在且可写
- 检查磁盘空间
- 确认视频 URL 有效

### 7.2 应急处理

#### 重启服务

```bash
# PM2
pm2 restart seedance2.0

# Docker
docker compose restart
```

#### 清理缓存

```bash
# 清理浏览器缓存
rm -rf data/cache/*

# 清理临时文件
rm -rf data/temp/*
```

## 9. 安全加固

### 9.1 基本安全措施

- ✅ 使用 HTTPS（Let's Encrypt）
- ✅ 修改默认管理员密码
- ✅ 限制 API 请求频率
- ✅ 定期更新依赖包
- ✅ 配置防火墙规则

### 9.2 防火墙配置

```bash
# Ubuntu UFW
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 22/tcp
sudo ufw enable
```

### 9.3 敏感信息保护

- `.env` 文件不要提交到版本控制
- SessionID 定期更新
- SMTP 密码使用授权码而非登录密码
