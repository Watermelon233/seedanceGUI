## 2. 认证与会话架构

```mermaid
sequenceDiagram
    participant U as 用户
    participant F as 前端
    participant B as 后端
    participant DB as 数据库
    participant JM as 即梦平台

    Note over U,JM: 双会话模型

    U->>F: 注册/登录
    F->>B: POST /api/auth/login
    B->>DB: 验证用户
    DB-->>B: 用户信息
    B-->>F: sessionId + user
    F->>F: 存入 localStorage
    Note over F: seedance_session_id<br/>seedance_user_cache

    F->>B: GET /api/auth/me<br/>Header: X-Session-ID
    B->>DB: 验证会话
    DB-->>B: 会话有效
    B-->>F: 当前用户

    Note over U,JM: 即梦 Session 管理

    F->>B: POST /api/settings/session-accounts
    B->>DB: 保存 jimeng_session_accounts

    F->>B: 视频生成请求
    B->>B: 解析生效 Session
    Note over B: user_default<br/>> legacy_global<br/>> env_default<br/>> none
    B->>JM: 使用解析的 Session
```

## 4. 数据流架构

### 4.1 单任务生成流程

```mermaid
sequenceDiagram
    participant U as 用户
    participant F as 前端
    participant B as 后端
    participant JM as 即梦平台
    participant FS as 文件系统

    U->>F: 上传图片、输入提示词
    F->>B: POST /api/generate-video<br/>multipart/form-data
    B->>FS: 保存图片到本地
    B->>JM: 上传图片到 ImageX
    JM-->>B: image_uri
    B->>B: 构建 draft task
    B->>B: 展开 output tasks
    B->>JM: 提交生成请求
    JM-->>B: history_id
    B-->>F: taskId + outputTaskIds

    loop 轮询
        F->>B: GET /api/task/:taskId
        B->>JM: 查询任务状态
        JM-->>B: status/result
        B-->>F: 任务状态
    end

    B->>FS: 保存视频到本地
    B->>B: 更新 download_status
    F->>B: POST /api/download/tasks/:id/file-token
    B-->>F: token
    F->>F: 触发浏览器下载
```

### 4.2 批量生成流程

```mermaid
sequenceDiagram
    participant U as 用户
    participant F as 前端
    participant BS as BatchScheduler
    participant VG as VideoGenerator
    participant JM as 即梦平台

    U->>F: 选择项目、选择任务
    F->>BS: POST /api/batch/generate
    BS->>BS: 验证 ownership
    BS->>BS: 检查 draft 类型
    BS->>BS: 展开 output tasks
    BS->>BS: 创建 batch 记录
    BS-->>F: batchId

    par 并发执行
        BS->>VG: 任务1
        BS->>VG: 任务2
        BS->>VG: 任务3
    end

    VG->>JM: 提交生成
    JM-->>VG: history_id
    VG->>BS: 更新状态

    BS-->>F: 批量状态更新
```

## 7. 下载链路架构

```mermaid
sequenceDiagram
    participant U as 用户
    participant F as 前端
    participant B as 后端
    participant FS as 本地文件系统
    participant JM as 即梦平台

    Note over U,JM: 第一阶段：服务端下载到本地

    B->>JM: 获取视频 URL
    JM-->>B: video_url
    B->>FS: 下载视频到本地
    B->>B: 更新 download_status=done
    B->>B: 保存 video_path

    Note over U,JM: 第二阶段：浏览器通过 token 下载

    F->>B: GET /api/download/tasks
    B-->>F: 下载任务列表

    U->>F: 点击下载
    F->>B: POST /api/download/tasks/:id/file-token
    B->>B: 生成一次性 token
    B-->>F: {token, filename}

    F->>F: 构造下载链接
    F->>B: GET /api/download/file-by-token?token=xxx
    B->>B: 验证 token + ownership
    B->>FS: 读取文件
    B-->>F: 文件流
    F->>U: 触发浏览器下载
```

## 11. 核心业务流程图

### 11.1 用户注册登录流程

```mermaid
flowchart TD
    START([开始]) --> CHOICE{操作类型}

    CHOICE -->|注册| R1[输入邮箱密码]
    R1 --> R2[获取邮箱验证码]
    R2 --> R3[提交注册信息]
    R3 --> R4{注册成功?}
    R4 -->|否| R1
    R4 -->|是| LOGIN

    CHOICE -->|登录| L1[输入邮箱密码]
    L1 --> L2{验证成功?}
    L2 -->|否| L1
    L2 -->|是| LOGIN[创建会话]

    LOGIN --> S1[保存 sessionId]
    S1 --> S2[保存 user cache]
    S2 --> S3[跳转到主页]

    style START fill:#e1f5fe
    style LOGIN fill:#c8e6c9
    style S3 fill:#c8e6c9
```

### 11.2 视频生成流程

```mermaid
flowchart TD
    START([开始]) --> AUTH{已登录?}
    AUTH -->|否| LOGIN[跳转登录]
    AUTH -->|是| SELECT[选择模型]

    SELECT --> UPLOAD[上传参考图 1-5张]
    UPLOAD --> PROMPT[输入提示词]
    PROMPT --> CONFIG[配置比例/时长]

    CONFIG --> CHECK{检查配置}
    CHECK -->|无图片| UPLOAD
    CHECK -->|无提示词| PROMPT
    CHECK -->|有 SessionID| SUBMIT[提交生成请求]

    SUBMIT --> EXPAND[展开 output 任务]
    EXPAND --> POLL[开始轮询]

    POLL --> STATUS{任务状态}
    STATUS -->|pending| POLL
    STATUS -->|generating| POLL
    STATUS -->|done| SAVE[保存视频]
    STATUS -->|error| ERROR[显示错误]

    SAVE --> DOWNLOAD[提供下载]
    DOWNLOAD --> END([结束])

    style START fill:#e1f5fe
    style END fill:#c8e6c9
    style ERROR fill:#ffcdd2
```
